# 📝 커밋 컨벤션

## 커밋 메시지 구조

```
타입: 제목

본문 (선택)

이슈 번호 (선택)
```

## 타입 종류

| 타입 | 설명 | 예시 |
|------|------|------|
| `feat` | 새로운 기능 추가 | feat: 회원 로그인 API 추가 |
| `fix` | 버그 수정 | fix: JWT 토큰 만료 오류 수정 |
| `docs` | 문서 수정 | docs: README 실행 방법 수정 |
| `style` | 코드 포맷 변경 (기능 변경 없음) | style: 코드 정렬 수정 |
| `refactor` | 코드 리팩토링 | refactor: 회원 서비스 로직 분리 |
| `test` | 테스트 코드 추가/수정 | test: 회원가입 단위 테스트 추가 |
| `chore` | 빌드 설정, 의존성 변경 | chore: Gradle 의존성 추가 |

## 작성 규칙

1. 제목은 **50자 이내**
2. 제목 끝에 **마침표 금지**
3. 제목은 **명령문** 형태로 (추가, 수정, 삭제)
4. 이슈가 있을 경우 **close #이슈번호** 로 자동 close

## 예시

```
feat: 회원 로그인 API 추가

- JWT Access Token, Refresh Token 발급
- Redis에 Refresh Token 저장

close #1
```

```
fix: 비밀번호 암호화 누락 버그 수정

close #5
```
